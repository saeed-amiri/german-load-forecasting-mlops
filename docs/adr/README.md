# Architecture Decision Records (ADR)

This folder stores short records of important technical and process decisions.

## Naming

- Use sequential numbers: `0001`, `0002`, `0003`, ...
- Use a short slug after the number.
- Example: `0001-adopt-adr-process.md`

## Status values

- Proposed
- Accepted
- Superseded
- Rejected

## Rules

- One ADR per decision.
- Keep ADRs short and focused.
- Never delete old ADRs; mark them as superseded.
- Link each ADR to the related Jira issue and PR.
