# Jira Baseline (Step 1)

This is the first delivery artifact for project management setup.

## Is this delivery or development?

- This is delivery/operations design, not product code development.
- It helps the team organize and track work before/while coding.

## Minimal workflow (v1)

Use these statuses:

1. Backlog
2. Ready
3. In Progress
4. In Review
5. Done

## Very small issue model (v1)

- Epic: large objective
- Story: user-facing value
- Task: technical implementation work
- Bug: defect to fix

## First operating rule

- Every code pull request should reference one issue.

## What happens to files afterwards?

- This file stays in the repository as process documentation.
- It can be extended, split, or replaced as your process matures.
- It does not affect runtime behavior of services.
