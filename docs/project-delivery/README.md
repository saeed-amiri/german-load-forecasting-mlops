# Project Delivery Guide

This folder contains the minimum delivery artifacts for managing work in Jira.

## What This Is

- Delivery process documentation.
- Not application runtime code.

## When To Use Each File

1. Baseline workflow:
   - Use `01-jira-baseline.md` to align on statuses and issue types.
2. Story work item:
   - Use `templates/story-template.md` for planned feature/value work.
3. Bug work item:
   - Use `templates/bug-template.md` for defects.
4. Incident work item:
   - Use `templates/incident-template.md` for production/service-impact events.

## Team Rule (Starter)

- Every pull request should reference one Jira issue.

## How This Evolves

- Keep the workflow simple first.
- Add fields only when they reduce confusion.
- Update templates from real team feedback every sprint.
